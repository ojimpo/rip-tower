# Rip Tower テストパターン

## テスト環境

- THE CHECKERS / ALL Song Request の Disc 1 は完了済み
- Disc 2 が手元にあるはず

---

## テスト1: 複数枚組 Disc 2（最優先）

**使うCD:** THE CHECKERS / ALL Song Request Disc 2（手元にあるもの）

**手順:**
1. Disc 2 をドライブに入れる
2. ダッシュボード → ▶ボタン
3. リッピング開始ダイアログで:
   - ソースタイプ: 図書館
   - Disc番号: **2**
   - 総Disc数: **2**
4. リッピング〜エンコード完了を待つ
5. review 画面でメタデータ確認
6. 承認

**確認ポイント:**
- [ ] トラックタイトルが正しく表示されるか
- [ ] メタデータの album_base が「ALL Song Request」（Disc suffix なし）か
- [ ] 承認後、`/mnt/media/music/THE CHECKERS/ALL Song Request [DISC2]/` に配置されるか
- [ ] Disc 1 と同じアーティスト名・アルバム名（base）になっているか
- [ ] FLAC タグの ALBUM が「ALL Song Request」（Disc suffix なし）、DISCNUMBER=2、TOTALDISCS=2 か

---

## テスト2: 手持ちCD（source_type: owned）

**使うCD:** 自分が持っているCD何でも。MusicBrainz に登録されてそうなメジャーなやつが望ましい

**手順:**
1. CD をドライブに入れる
2. ▶ボタン → ソースタイプ: **手持ち**
3. 完了まで待つ

**確認ポイント:**
- [ ] source_type が「手持ち」として履歴に表示されるか
- [ ] MusicBrainz disc ID でメタデータが取れるか（confidence 高めになるはず）
- [ ] confidence 85 以上なら自動承認されるか
- [ ] 自動承認された場合、Discord に完了通知が来るか（URL が `http://rip-tower.arigato-nas/job/xxx` の形式か）
- [ ] ファイルが正しいフォルダに配置されるか

---

## テスト3: 図書館CD + kashidashi マッチ

**借りてくるCD:** 葛飾図書館で借りられるCD何でも。kashidashi に登録されるもの

**手順:**
1. kashidashi で貸出処理をする（通常通り）
2. CD をドライブに入れる
3. ▶ボタン → ソースタイプ: **図書館**
4. 完了まで待つ
5. review 画面で kashidashi タブを確認

**確認ポイント:**
- [ ] kashidashi タブにマッチ候補が表示されるか
- [ ] disc ID 完全一致があれば confidence が高いか
- [ ] マッチを確定できるか
- [ ] 履歴画面で「図書館 kashidashi #xxx」と表示されるか

---

## テスト4: アートワーク取得

**使うCD:** テスト2 or 3 のどれかで確認

**手順:**
1. リッピング完了後、ジョブ詳細 → アートワークタブを開く

**確認ポイント:**
- [ ] Cover Art Archive or iTunes からアートワーク候補が表示されるか
- [ ] 選択できるか
- [ ] 承認後、フォルダに cover.jpg が配置されるか
- [ ] FLAC にアートワークが埋め込まれるか（`metaflac --list file.flac | grep PICTURE` で確認）

---

## テスト5: 歌詞取得

**使うCD:** 邦楽のメジャーなアルバム（LRCLIB に歌詞がありそうなもの）。椎名林檎、BUMP OF CHICKEN あたりは LRCLIB にある可能性が高い

**手順:**
1. リッピング完了後、ジョブ詳細 → 歌詞タブを開く

**確認ポイント:**
- [ ] 自動で歌詞が取得されているか（LRCLIB）
- [ ] 同期歌詞（LRC）がある場合、タイムスタンプ付きで表示されるか
- [ ] 「取得」ボタンで個別トラックの歌詞を再取得できるか
- [ ] FLAC タグに歌詞が埋め込まれるか

---

## テスト6: メタデータ編集 → 再反映（complete 後）

**使うCD:** テスト1〜3 で完了済みのジョブを使う

**手順:**
1. 完了済みジョブの詳細画面を開く
2. アーティスト名やトラックタイトルを編集
3. 「再反映」ボタン（apply）を押す

**確認ポイント:**
- [ ] 編集した内容が FLAC タグに反映されるか
- [ ] アーティスト名を変えた場合、フォルダがリネームされるか

---

## テスト7: review リマインド通知

**手順:**
1. confidence が低いCDをリッピングする（テスト3 でマイナーなCDを使う）
2. review 状態のまま放置する（6時間後にリマインドが飛ぶ設定）

**確認ポイント:**
- [ ] Discord にリマインド通知が来るか
- ※ 6時間待つのが大変なら、設定画面で reminder_initial_hours を 0.01（約36秒）に一時的に変更してテスト

---

## テスト8: エラーリカバリ

**使うCD:** 傷があるCD、または途中でイジェクトする

**手順:**
1. リッピング中にCDをイジェクト（or 傷CDを使う）
2. エラーになることを確認
3. ダッシュボードの「要対応」セクションに表示されるか確認
4. 再リッピングを試みる

**確認ポイント:**
- [ ] エラーステータスが正しく表示されるか
- [ ] Discord にエラー通知が来るか
- [ ] 失敗トラックが赤く表示されるか
- [ ] 「失敗トラックを再リップ」ボタンが機能するか

---

## テスト9: WAV インポート（hachiman-desk 連携）

**手順:**
1. 何らかのWAVファイルを用意（Windows の EAC 等でリップしたもの、なければテスト用に適当な WAV）
2. ダッシュボードの上部アップロードアイコンから Import 画面へ
3. WAV ファイルをドラッグ&ドロップ
4. アーティスト・アルバムのヒントを入力
5. 送信

**確認ポイント:**
- [ ] WAV がアップロードされてジョブが作成されるか
- [ ] メタデータ解決が走るか（disc ID なしのテキスト検索）
- [ ] エンコード → finalize まで完了するか

---

## 優先度

1. **テスト1**（Disc 2）— すぐできる、album_group の動作確認
2. **テスト2**（手持ちCD）— 自動承認の動作確認
3. **テスト3**（図書館CD）— kashidashi 連携確認
4. **テスト4-5**（アートワーク・歌詞）— テスト2-3 と同時に確認可能
5. **テスト6**（再反映）— 完了済みジョブで確認
6. **テスト7-9** — 余裕があれば
